## Task Scheduling & Notification Platform

Backend: Laravel 12 (PHP 8.2+), Sanctum, MySQL/SQLite
Frontend: Vue 3 + TypeScript (Vite)

### Local Development

Backend
- `composer install`
- `copy .env.example .env`
- Set `DB_CONNECTION=sqlite` in `.env`
- `php artisan key:generate`
- `php artisan migrate --force`
- `php artisan db:seed --force`
- `php artisan serve`

Frontend
- `npm i`
- `npm run dev`

### Features
- Tasks CRUD, reassign, status updates, search and filters
- Strict no-overlap assignment rule
- Availability consolidation job
- Token auth with Sanctum
- Simple admin UI: login, board (Kanban), create/reassign modals

### Seeded Data
- Users: 10 random + Admin `admin@example.com` / `password`
- Statuses: `todo`, `in_progress`, `done`, `blocked`

### Notes / Assumptions
- Overlap: any partial intersection (same-day/time) is blocked
- Availability: consolidated from tasks via background job
- Notifications: simulated via queued log entries
